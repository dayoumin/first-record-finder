# 추가 데이터소스 검토

## 현재 WoRMS로 해결되는 것
- ✅ 해양생물 학명 검증
- ✅ 이명(synonym) 목록 추출
- ✅ 분류 정보 (Kingdom ~ Genus)
- ✅ 원기재 정보

---

## 추가로 필요할 수 있는 데이터소스

### 🔴 높음 (권장)

| 데이터소스 | 용도 | species_checker 코드 |
|-----------|------|----------------------|
| **한국해양생물목록 (KORIB)** | 국명 조회, 기존 최초기록 참조 | - |
| **국립생물자원관 DB (NIBR)** | 한국 표본 정보, 채집 기록 | `nibrValidator.ts` 참고 가능 |

**KORIB 연동 시 장점:**
- 기존에 정리된 최초기록 정보 참조 가능
- 국명 자동 조회
- 작업량 대폭 감소

### 🟡 중간

| 데이터소스 | 용도 | 비고 |
|-----------|------|------|
| **KCI Open API** | 한국 논문 자동 검색 | 무료 API 제공 |
| **CrossRef/DOI** | 문헌 메타데이터 자동 추출 | DOI 기반 조회 |
| **RISS API** | 학위논문 검색 | 회원가입 필요 |

### 🟢 낮음

| 데이터소스 | 용도 | 비고 |
|-----------|------|------|
| **GBIF** | 글로벌 채집 기록 (한국 포함) | 이미 정리된 데이터 |
| **FishBase** | 어류 추가 정보 | 어류 한정 |
| **SeaLifeBase** | 비어류 해양생물 | 참고용 |

## species_checker에서 재사용 가능한 코드

### 직접 재사용 가능

| 파일 | 용도 | 재사용 방법 |
|------|------|------------|
| `fetchWithRetry.ts` | API 재시도 로직 | ✅ 이미 복사함 |
| `wormsValidator.ts` | WoRMS API 호출 | 참고용 (이미 구현) |

### 참고용

| 파일 | 용도 | 참고 포인트 |
|------|------|------------|
| `nibrValidator.ts` | 국립생물자원관 연동 | 엑셀 파일 처리 로직 |
| `colValidator.ts` | CoL API 연동 | 대량 데이터 처리 |
| `utils.ts` | 학명 정규화 | `normalizeName()` 함수 |
| `name-preprocessing.ts` | 학명 전처리 | 약어 확장 등 |

### 참고할 문서

| 문서 | 내용 |
|------|------|
| `docs/DATA_MANAGEMENT.md` | 오프라인 데이터 관리 |
| `docs/TYPO_DETECTION_*.md` | 오타 검출 시스템 |

## 연동 우선순위 권장

```
1단계 (현재): WoRMS만 사용
     ↓
2단계: KORIB 연동 (국명 + 기존 최초기록)
     ↓
3단계: KCI API 연동 (한국 논문 자동 검색)
     ↓
4단계: NIBR 연동 (표본 정보)
```
